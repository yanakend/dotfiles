eio_mkdir

(PECL eio >= 0.0.1dev)

eio_mkdir — Create directory

説明

resource eio_mkdir ( string $path , int $mode [, int $pri = EIO_PRI_DEFAULT [, mixed $callback = NULL [, mixed $data = NULL ]]] )

eio_mkdir() creates directory with specified access mode.

パラメータ

path

Path for the new directory.

mode

Access mode, e.g. 0755

pri

リクエストの優先順位。EIO_PRI_DEFAULT、EIO_PRI_MIN、EIO_PRI_MAX あるいは NULL を指定します。 NULL を渡した場合、pri は内部的に EIO_PRI_DEFAULT となります。

callback

callback 関数が、リクエスト終了時にコールされます。 この関数のプロトタイプは、

void callback($data, $result);
に一致しなければなりません。 $data は、リクエストに渡すカスタムデータです。 $result にはリクエスト固有の結果が格納されます。基本的には、対応するシステムコールが返すそのままの値となります。

data

Arbitrary variable passed to callback.

返り値

eio_mkdir() returns request resource on success or FALSE on error.

例

例1 eio_mkdir() example

<?php
$temp_dirname = "eio-temp-dir";

/* Is called when eio_mkdir() finishes */
function my_mkdir_callback($data, $result) {
 if ($result == 0 && is_dir($temp_dirname)
   && !is_readable($temp_dirname)
   && is_writable($temp_dirname)) {
  echo "eio_mkdir_ok";
 }

 // Remove directory
    if (file_exists($data))
        rmdir($temp_dirname);
}

// Create directory with access mode 0300
eio_mkdir($temp_dirname, 0300, EIO_PRI_DEFAULT, "my_mkdir_callback", $temp_dirname);
eio_event_loop();
?>

上の例の出力は、 たとえば以下のようになります。

eio_mkdir_ok
